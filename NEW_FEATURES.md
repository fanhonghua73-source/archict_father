# 新功能实现说明

## 已完成的功能

### 1. 文件上传自动分类 ✅

**功能说明**：
当用户选择文件后，系统会根据文件名和文件类型自动建议分类。

**分类规则**：
- **效果图**：文件名包含"效果图"、"渲染"、"透视"、"鸟瞰"，或文件类型为 JPG/PNG
- **红线图**：文件名包含"红线"
- **测绘图**：文件名包含"测绘"、"地形"
- **地质报告**：文件名包含"地质"、"勘察"
- **合同**：文件名包含"合同"、"协议"
- **会议纪要**：文件名包含"会议"、"纪要"
- **进度计划**：文件名包含"进度"、"计划"
- **方案设计**：DWG文件且包含"平面"、"立面"、"剖面"
- **工作文件**：SKP文件

**使用方式**：
1. 点击"📤 上传"按钮
2. 选择文件
3. 系统会自动检测文件类型并填充
4. 如果检测到文件应该属于其他分类，会显示提示："建议分类: xxx"

- 忙碌度：0-100%，随机生成，可编辑

- 时间安排：可编辑（多行文本）

**操作方式**：

#### 激活/停用人员
- 点击人员格子（不是编辑按钮）
- 激活的人员：绿色边框，浅绿背景
- 停用的人员：灰色边框，白色背景

1. 点击人员格子右上角的 ✎ 按钮
2. 弹出编辑弹窗
3. 可以编辑：
   - 姓名
   - 忙碌度（滑块 0-100%）
   - 角色
   - 时间安排（多行文本）
4. 点击"保存"按钮

#### 查看人员状态
- **中列人员格子**：显示姓名和忙碌度条
- **右列时间规划**：显示激活人员的详细信息（姓名、角色、时间安排）

**时间安排格式示例**：

### 4. 新增项目功能 ✅

**功能说明**：
在项目列表页面添加了"新增项目"卡片，点击可以创建新项目。

**使用方式**：
1. 进入项目页面（首页）
2. 点击第一个"+ 新增项目"卡片
3. 填写项目信息：
   - 项目名称（必填）
   - 项目地点（必填）
   - 项目阶段（下拉选择：方案设计、初步设计、施工图设计、概念设计）
   - 总面积（数字）
   - 项目状态（下拉选择：设计中、施工中、规划中、已完成）
4. 点击"创建项目"按钮
5. 创建成功后，项目列表会自动刷新

**新增项目卡片样式**：
- 虚线边框
- "+" 图标
- "新增项目"文字
- 鼠标悬停时变绿色

## 数据持久化

所有数据都会保存到后端：

### 文件数据
- 位置：`backend/data/management/project_{id}.json`
- 字段：`files` 数组

### 人员数据
- 位置：`backend/data/management/project_{id}.json`
- 字段：`staff` 数组
- 修改后自动保存到后端

### 项目数据
- 位置：`backend/data/projects.json`
- 新增项目会自动分配 ID

## 技术实现

### 文件自动分类
```javascript
suggestCategory(fileName, fileType) {
    const name = fileName.toLowerCase()
    // 效果图
    if (name.includes('效果图') || (fileType === 'JPG' || fileType === 'PNG')) {
        uni.showToast({ title: '建议分类: 效果图', icon: 'none' })
    }
    // ... 其他分类规则
}
```

## 注意事项

1. **文件上传**：目前是虚拟上传，实际文件不会保存到磁盘，只保存文件记录
2. **人员数据**：修改后会立即保存到后端
3. **项目数据**：新增项目后会自动分配 ID
4. **数据隔离**：每个项目的文件和人员数据是独立的
